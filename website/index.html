Why Never Forget Tasks?

Purpose-built for CEO agents that manage teams of AI workers.

📋

Task Assignment

CEO agent assigns tasks to workers with titles, descriptions, deadlines, and clear ownership.

🔗

Dependencies & Pipelines

Express task chains with predecessor/successor IDs. Build DAGs for complex multi-stage workflows.

📊

Progress Reports

One-call summary of overdue, blocked, failed, and in-progress tasks — perfect for periodic CEO check-ins.

🔒

Permission Control

Only the assigned agent can update their own task. Prevents accidental or unauthorized changes.

⚠️

Failure Tracking

Workers must explain why tasks failed or got blocked. Reasons surface in CEO reports for fast follow-up.

🗄️

SQLite Persistence

Single-file database, zero config. Multiple agents and processes can share the same task store.

Quick Start

Up and running in under a minute.

# Install and run npm install @miguoliang/openclaw-never-forget-tasks npm run build && node dist/mcp-server.js # Add to OpenClaw's MCP config { "mcp": { "servers": { "never-forget-tasks": { "command": "node", "args": ["node_modules/@miguoliang/openclaw-never-forget-tasks/dist/mcp-server.js"] } } } }
# Clone, build, and install as OpenClaw plugin git clone https://github.com/miguoliang/openclaw-never-forget-tasks.git cd openclaw-never-forget-tasks npm install && npm run build cd plugin-openclaw && npm install # Link and enable openclaw plugins install --link ./plugin-openclaw openclaw plugins enable never-forget-tasks
import { TaskStore } from "@miguoliang/openclaw-never-forget-tasks/store"; import { formatReportForAgent } from "@miguoliang/openclaw-never-forget-tasks/report"; const store = new TaskStore(); // Assign a task const task = store.assign({ assignee: "agent_research", title: "Complete market research", assigned_by: "ceo_agent", }); // Get progress report console.log(formatReportForAgent(store, { language: "en" }));

Available Tools

7 tools exposed via MCP and OpenClaw Plugin.

ToolDescription
task_assignAssign a task to an agent with title, description, deadline, and dependencies
task_update_statusUpdate status (only assignee can update; status_note required for blocked/failed)
task_list_by_assigneeList tasks for a specific agent, optionally filtered by status
task_get_progress_reportFull progress summary — overdue, blocked, failed, by assignee
task_list_overdueList all overdue incomplete tasks
task_list_blockedList tasks blocked by incomplete predecessors
task_getGet full task details by ID
